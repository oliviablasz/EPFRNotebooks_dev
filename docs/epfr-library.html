2 EPFR Library

The EPFR Library has an extensive list of applicable functions for use on our data and will be used in this demonstration of our strategies.

To install the library EPFR, click here to download zip from github.

Before installing the library EPFR, please install the dependencies this library requires: devtools, RODBC, and RDCOMClient. You can find the code to do this below.

install.packages("RDCOMClient", repos = "http://www.omegahat.net/R")

x <- c("devtools", "RODBC", "RDCOMClient")
w <- !is.element(x, dimnames(installed.packages())[[1]])
if (any(w)) for (y in x[w]) install.packages(y)

Now you can properly install and load the library EPFR.

library("devtools")
install_github("vsrimurthy/EPFR")

library("EPFR")

2.2 ftp.get()

In this demonstration we will use the function ftp.get() from library('EPFR) to access files from EPFR’s ftp which requires the RCurl library.

library(RCurl)
## Warning: package 'RCurl' was built under R version 4.1.2

To use this function you will set a file path to the csv file that will be used for your strategy and a local folder that this file will be downloaded to:

file <- "EPFRHistory/Strategies/daily/FloPctCtry-daily.csv" # FTP FILE - FloPctCtry-daily.csv
fldr <- "C:\\temp" # LOCAL FOLDER

Next you will enter your username and password:

usr <- "YOUR USER NAME" # FTP USERNAME
pwd <- "YOUR PASSWORD" #FTP PASSWORD

Now you can call ftp.get() with the following inputs:

ftp.get(file, fldr, "filetransport.epfrglobal.com", usr, pwd, "ftp") # DOWNLOADS FILE FROM FTP TO LOCAL FOLDER, USING CREDENTIALS

After the file is downloaded the flow file’s local path can be stored with follow line:

flow.file <- paste(fldr, ftp.file(file), sep = "\\") # STORES FLOW FILE PATH

2.3 compound.flows()

EPFR’s function compound.flows() is very simple but quickly compounds our percentage flow file over a trailing lookback period.

compound.flows("DailyPercentageFlow.csv", lookback = 20)

2.4 bbk()

EPFR’s function bbk() is more complex as it creates our standard output model.

This function requires many different parameters such as our compounded daily percentage flow file and a return file.

x <- "Compounded Percentage Flow"
y <- "Return's"
hz <- "Return horizon"
nBin <- "Number of bins (to sort)"
doW <- "Day of the week to trade on"
delay <- "Delay in getting the data"
idx <- "Universe"

Please refer to the library documentation for the complete list of parameters of this function.

bbk(x, y, 1, hz[1], nBin, doW, T, 0, delay, idx)
